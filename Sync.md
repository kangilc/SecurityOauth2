  | 종류 | 설명 | 장점 | 단점 | 주요 사용 사례 |
  | :--- | :--- | :--- | :--- | :--- |
  | 실시간 (Real-time) | 원본 데이터가 변경되는 즉시, 거의 지연 없이 동기화됩니다. (보통 이벤트 기반 아키텍처, 메시지 큐 등을 사용) | - 데이터 정합성이 가장 높음<br>- 사용자가 항상 최신 데이터 확인 가능 | - 시스템 부하가 큼<br>- 구현이 복잡하고 비용이 높음 | - 은행 거래 시스템<br>- 실시간 재고 관리<br>- 항공권 예약 시스템 |
  | 준-실시간 (Near Real-time) | 수 초에서 수 분 사이의 매우 짧은 주기로 동기화됩니다. (마이크로 배치 방식) | - 실시간에 가까운 데이터 제공<br>- 시스템 부하를 어느 정도 제어 가능 | - 실시간보다는 약간의 지연 발생<br>- 구현이 비교적 복잡함 | - 모니터링 대시보드<br>- 실시간 로그 분석<br>- 위치 기반 서비스 |
  | 배치 (Batch) | 정해진 스케줄(시간, 일, 주, 월 등)에 따라 주기적으로 동기화됩니다. 가장 전통적이고 일반적인 방식입니다. | - 시스템 부하 예측 및 관리가 용이<br>- 대용량 데이터 처리에 효율적<br>- 구현 및 운영이 비교적 단순 | - 데이터 지연(Latency) 발생<br>- 실시간성이 중요한 업무에는 부적합 | - (매일) 야간 ETL 작업, 데이터 웨어하우스<br>- (매시간) 통계 리포트 생성<br>- (매월) 월말 정산 및 빌링 |
  | 수동 (Manual) | 사용자가 직접 '동기화' 버튼을 누르거나 특정 액션을 취했을 때만 동기화가 실행됩니다. | - 사용자가 원하는 시점에만 실행<br>- 불필요한 시스템 리소스 낭비 없음 | - 자동화 불가<br>-사용자가 잊으면 데이터 불일치 발생 | - 스마트폰 주소록 동기화<br>- 클라우드 스토리지 파일 동기화 |

  ### 주기 결정 시 핵심 고려사항
  1. 데이터의 신선도(Freshness) 요구 수준: 데이터가 얼마나 최신 상태로 유지되어야 하는가? (가장 중요한 요소)

### '테이블 동기화 기능' 개발 일정 제안
  | 단계 | 주요 활동 (Tasks) | 예상 기간 | 산출물 / 완료 조건 |
  | :--- | :--- | :--- | :--- |
  | 1. 분석 및 설계 | - 요구사항 확정: 동기화 방향(단방향/양방향), 주기, 대상 테이블 최종 확정<br>- 아키텍처 설계: 스크립트 실행 방식(수동/스케줄러), DB 연결 방식 설계<br>- 예외 처리 및 로깅 정책 
  설계: 오류 발생 시 처리 방안, 실행 결과 기록 방식 정의<br>- 기술 설계서 작성 | 1주 | - 최종 요구사항 명세서<br>- 기술 설계서 |
  | 2. 개발 | - DB 연결 모듈 개발: STG/PRD DB 접속 및 자격 증명 관리 기능<br>- 동기화 코어 로직 개발: 데이터 비교 및 동기화 실행 로직<br>- 설정 기능 개발: 동기화할 테이블, 방향 등을 외부 설정 파일로
   분리<br>- 로깅 및 알림 기능 개발: 실행 결과 및 오류 로그 기록, 실패 시 알림 기능 | 2-3주 | - 개발 완료된 동기화 스크립트<br>- 단위 테스트 완료 |
  | 3. 테스트 | - 단위/통합 테스트: 개발 환경에서 기능별, 전체 흐름 테스트<br>- QA 테스트: QA(품질보증)팀 또는 동료 개발자가 테스트 환경에서 기능 및 성능 테스트<br>- 사용자 인수 테스트(UAT): 실제
  운영 담당자가 사용 시나리오에 맞춰 최종 검수 | 1-2주 | - 테스트 케이스<br>- QA 및 UAT 통과 보고서 |
  | 4. 배포 | - 배포 스크립트 작성: 실제 운영 환경에 배포하기 위한 스크립트 준비<br>- 운영 메뉴얼(Runbook) 작성: 스크립트 실행 방법, 모니터링, 장애 조치 가이드<br>- 운영 환경 배포 및 최종 점검 |
  1-2일 | - 배포 스크립트<br>- 운영 메뉴얼 |
  | 5. 안정화 | - 초기 모니터링: 배포 후 동기화 기능이 안정적으로 동작하는지 집중 모니터링<br>- 긴급 버그 수정: 배포 초기 발생할 수 있는 긴급 장애 대응 및 핫픽스(Hot-fix) | 1주 | - 안정화 보고서 |
